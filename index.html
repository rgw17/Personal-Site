Synthetic Dataset Generator (ongoing)

Stack: Python, Sklearn, Pandas, Unity, C#

Introduction: A common problem with developing machine learning algorithms is finding enough training data. Companies like Scale AI have entered this space, providing high quality, labeled images for algorithms to learn. It is often difficult to collect such data from the real world (ex: images of the street for a self-driving car).

Solution: I am building a system that automatically generates and tags images. The images are generated from Unity. Unity allows you to adjust a variety of features in 3D objects, such as lighting, shading, texture, color, size, etc. providing virtually unlimited training data. Once provided a base object (such as a car), the program will generate hundreds of images of the 3D car model in different lighting and background scenarios. These images will be fed through a pipeline and labeled with Python, using ML libraries to detect and label objects.

Instagram Spam Detector

Stack: Python, NumPy

Introduction: I made this project to apply some knowledge I learned from my Artificial Intelligence class at USC. After browsing Kaggle, I found a good Instagram account dataset to practice on. Each row of the CSV file contains account information such as # of followers, profile picture (or not), etc.

Solution: I implemented Naive Bayes Classifier, a popular machine learning algorithm that applies Bayes' theorem. Coding the algorithm from scratch, it multiples each subprobability of each feature, then chooses the label with the highest probability.

Challenges I ran into were continuous features such as # of followers. Unlike discrete features, continuous ones can have an infinute number of values. To solve this, I created a density function using Gaussian distribution to calculate how many standard deviations a given feature was from the median.

Another problem I ran into were unique features with a probability of 0. Because all subprobabilities are multiplied together, multiplying by 0 could void the entire product. Therefore, I used Laplace smoothing, which "smoothes" to produce a nonzero probability.

Using cross-validation, I was able to predict whether an Instagram account was spam or not with an average of 88% accuracy, and a high of 90.33% accuracy. I used NumPy to shuffle tables and parse the CSV.



Stock Portfolio Manager

Stack:
Frontend: HTML/CSS/Bootstrap, Javascript
Backend: Java, Firebase
Testing: Java, Selenium, Mockito, JUnit

Introduction: As part of my software engineering group project, I created a stock portfolio management website. I mostly worked on the backend, specifically Java servlets, Firebase authentication, session management, and backend testing with JUnit/Mockito. I also made minor changes to the frontend, implementing a CSV parsing button. Users can upload CSV files containing stock information to add it to their portfolio, or manually enter it instead. Lastly, I created Selenium/Cucumber frontend tests.

Features:
  • account creation
  • 120 second inactivity logout
  • 3x consecutive invalid login lockout
  • CSV parsing
  • watchlist and portfolio tables
  • stock history visualization
  • backend testing w/ JUnit & Mockito
  • frontend testing w/ Selenium & Cucumber
